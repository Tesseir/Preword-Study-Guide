Prework Study Guide

HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • h1, h2, h3, h4, h5, h6 = header leveling
  • ul ol li = unorder list, ordered list, and listed items
  • img = image, src attribute defines location of image, alt attribute defines purpose and describes image
  • br = empty line or space between 2 blocks of content, think "bracket" but invis
  • a = anchor, href = attribute point to url
  • Link: creates connection to external file, rel: attribute specifies relationship between current and linked docs

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Height: dictates how many pixels tall something is.
  • Width: dictates how many pixels wide something is.
  • Auto: dictates value to left and right.
  • Border: assigns border with width of xyz.
  • Box shaddow: adds shaddow affect around frame of element.
  • Period preceding X designates X as class selector. Any html class X will gain property assignments from Css.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git merge: merges branches
  • git branch: lets you create list rename and delete branches
  • git commit: snapshot of your repo at this point and time
  • git fetch: asks origin (GitHub) for a list of changes. It replies back with new branches and commits.
  • git pull: Git pull retrieves the changes from origin.
  • git push:Git push puts your branches and commits on origin.
